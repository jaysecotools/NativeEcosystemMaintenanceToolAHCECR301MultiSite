EcoMaintain Multi-Site User Guide

Launch App

Overview

EcoMaintain is a comprehensive web-based application designed to help environmental professionals manage native ecosystem maintenance activities across multiple sites in compliance with AHCECR301 requirements. The multi-site version allows you to manage an unlimited number of separate conservation areas, each with their own assessments, species records, maintenance activities, and monitoring data.

Multi-Site Support

Manage multiple conservation areas independently with easy switching between sites

AHCECR301 Compliant

Built specifically to meet the requirements of ecosystem maintenance competency

Offline Capable

All data stored locally - no internet connection required for core functionality

Key Features
  • Multi-site management - Create and switch between unlimited sites
  • Site assessment - Record ecosystem health and threats
  • Maintenance planning - Schedule and track activities
  • Species inventory - Catalog flora and fauna
  • Monitoring - Record field observations and surveys
  • Compliance tracking - Monitor regulatory requirements
  • Reporting - Generate PDF, Excel, and Word reports
  • Interactive mapping - Visualize site zones and features
System Requirements
  • Modern web browser (Chrome, Firefox, Safari, Edge)
  • JavaScript enabled
  • Recommended screen resolution: 1280x768 or higher
  • Internet connection only required for:
    • Map tiles (OpenStreetMap)
    • External libraries (Bootstrap, Chart.js, etc.)

Multi-Site Management New Feature

The multi-site feature allows you to manage multiple conservation areas, properties, or project sites independently. Each site maintains its own separate data while sharing the same application interface.

Key Multi-Site Benefits
  • Complete Isolation: Data from different sites never mixes
  • Easy Switching: Change between sites with a single click
  • Unlimited Sites: Create as many sites as you need
  • Site-Specific Reporting: Reports automatically filter to current site
  • Consistent Templates: All sites use the same assessment and monitoring forms

Managing Sites

Adding a New Site

1 Navigate to Settings → Site Management

2 Click "Add New Site" button

3 Enter site details:

  • Site Name (required)
  • Location description
  • Area in hectares
  • Detailed description

4 Click "Save Site"

Tip: The first site you create becomes your current site automatically. Add descriptive names that clearly identify each location.
Switching Between Sites

There are two ways to switch sites:

Method 1: Site Selector

  • Located at the top of the sidebar
  • Click the dropdown and select any site
  • Confirm the switch in the popup

Method 2: Manage Sites Modal

  • Go to Settings → Manage Sites
  • Click the button next to any site
  • Confirm to switch immediately

Editing and Deleting Sites

Edit Site Details

1 Open Manage Sites modal

2 Click the button next to a site

3 Update any information

4 Click "Update Site"

Changes apply immediately and are reflected throughout the application.

Delete a Site

1 Open Manage Sites modal

2 Click the button

3 Confirm deletion

Warning: Deleting a site permanently removes ALL associated data (assessments, species, activities, monitoring). This action cannot be undone.

Site Identification

The current site is always clearly indicated throughout the application:

  • Site Badge: Colored badge appears next to section titles
  • Site Selector: Shows current site name in the sidebar
  • Dashboard Header: Displays current site name prominently
  • All Data Views: Tables and cards only show data for current site
Best Practice: Before entering new data, always check the site selector to ensure you're working in the correct site. The site badge provides a quick visual confirmation.

Dashboard

The dashboard provides a site-specific overview of your ecosystem management activities. All data shown is filtered to the currently selected site.

Site Context

The dashboard header shows which site you're currently viewing. All statistics and widgets display data only from this site.

Key Components
  • Statistics Panel: Upcoming tasks, assessments, species, compliance (site-specific)
  • Upcoming Tasks: Scheduled activities for current site
  • Site Health Indicators: Visual health metrics for current site
  • Recent Observations: Latest monitoring records for current site
  • Interactive Map: Geographic view with site-specific zone overlays
  • Compliance Checklist: Global compliance tracking (same across all sites)
Using the Dashboard

The dashboard helps you:

  • Quickly assess current priorities for each site
  • Monitor compliance progress globally
  • Access recent site-specific activities
  • Switch between sites to compare status
  • Export or print site-specific dashboard views
Tip: Use the export function to create quick status reports for individual sites. The dashboard automatically includes the site name in exports.

Site Assessment

Conduct and record comprehensive site assessments for each of your management areas. All assessments are automatically linked to the current site.

Site-Specific Assessments

When you create a new assessment, it's automatically assigned to the currently selected site. The site badge confirms which site you're assessing.

Creating a New Assessment

1 Verify you're in the correct site using the site selector or badge

2 Click the "New Assessment" button

3 Complete the assessment form:

  • Assessment date and assessor name
  • Site description and conditions
  • Ecosystem components present (flora, fauna, soil, water, etc.)
  • Overall ecosystem health rating
  • Identified threats
  • Supporting photos (optional)

4 Save the assessment

Viewing Assessments by Site

The assessments table only shows records for the current site:

  • Switch sites to view assessments from different locations
  • Click "View" to see detailed assessment information
  • Compare assessments over time for the same site
  • Export site-specific assessment reports
Best Practice: Conduct assessments quarterly or after significant events. Use consistent methodology for comparable results across different sites.

Maintenance Plan

Schedule, track, and manage maintenance activities for each site independently. Activities are always associated with the site where they were created.

Creating Site-Specific Activities

1 Ensure you're in the correct site

2 Click "New Activity"

3 Fill in activity details:

  • Activity type (weed control, revegetation, erosion control, etc.)
  • Title and description
  • Start and end dates
  • Location within the site
  • Assigned personnel
  • Priority level
  • Initial status

4 Save the activity

Viewing Activities by Site

Activities are organized in three tabs, all filtered to current site:

  • Upcoming: Planned and in-progress activities
  • Completed: Finished activities
  • Overdue: Past-due activities

Calendar View

The maintenance calendar shows activities for the current site only:

  • Color-coded by activity type
  • Click events for details
  • Switch between month, week, and day views
  • Calendar updates automatically when switching sites
Tip: Use the calendar to identify scheduling conflicts across different activities within the same site. The site selector lets you check schedules for different locations.

Species Register

Maintain comprehensive species inventories for each site. Species records are site-specific, allowing you to track different species compositions across your management areas.

Adding Site-Specific Species

1 Select the correct site

2 Click "New Species"

3 Enter species information:

  • Scientific name (required) and common name
  • Type (flora, fauna, fungi, other)
  • Conservation status
  • Description and habitat
  • Last observation date
  • Images for identification

4 Save the species record

Viewing Species by Site

Species can be viewed in two formats, both filtered to current site:

  • Card View: Visual presentation with images, ideal for field reference
  • Table View: Comprehensive list with filtering and sorting

Site-Specific Dropdowns

Species dropdowns in monitoring forms automatically show only species from the current site, ensuring data consistency:

  • Monitoring species selection
  • Fauna sighting species selection
  • Vegetation survey species selection
Best Practice: Record species as you encounter them during assessments and monitoring. Update last observed dates regularly. Use consistent naming conventions across all sites.

Vegetation Surveys

Data Management

Understand how EcoMaintain stores and manages your multi-site data.

Storage Architecture

  • All data stored locally in browser's localStorage
  • Each site's data is stored with site ID references
  • No server connection required - completely offline capable
  • Data persists between sessions on the same device

Backup and Export

To prevent data loss across all sites:

  • Export All Data: Creates a JSON backup of ALL sites and data
  • Backup Database: Timestamped backup for recovery
  • Individual Exports: Export site-specific data from each section

Data Integrity

Multi-site data integrity features:

  • All records automatically tagged with site ID
  • Form validation ensures required fields complete
  • Site-specific dropdowns prevent cross-site data mixing
  • Data relationships maintained within each site
Warning: The "Reset Test Data" function will delete ALL data across ALL sites. Always export a backup before using this feature.
Tip: Schedule regular exports (weekly/monthly) and store backups securely. This protects against accidental data loss or browser storage clearing.

Multi-Site Best Practices

Site Organization

  • Naming Convention: Use consistent naming (e.g., "Blue Mountains - North", "Blue Mountains - South")
  • Site Boundaries: Clearly define site areas to avoid data overlap
  • Hierarchy: Create separate sites for distinct management areas, not sub-zones

Data Entry Workflow

  • Verify Site First: Always check site selector before entering data
  • Batch Processing: Complete all entries for one site before switching
  • Cross-Site Reference: Use reports to compare data across sites

Monitoring Consistency

  • Use the same assessment methods across all sites for comparability
  • Standardize threat categories and health ratings
  • Maintain consistent species naming conventions

Reporting Strategy

  • Generate site-specific reports for individual management reviews
  • Use compliance reports for organization-wide tracking
  • Archive reports by site and date for historical reference
Pro Tip: Create a "master site" template with standard species and activity types, then duplicate by entering similar data in new sites. This ensures consistency across your organization.

Frequently Asked Questions

There's no technical limit to the number of sites you can create. The only constraint is your browser's localStorage capacity (typically 5-10MB). Each site with moderate data uses about 50-100KB, so you can manage 50-100 sites comfortably.

Data is intentionally isolated between sites to maintain integrity. However, you can use species information as a reference when adding similar species to new sites. For organization-wide standards, maintain a separate reference document.

All data associated with that site (assessments, species, activities, monitoring, reports) is permanently deleted. This action cannot be undone. Always export a backup before deleting important sites.

There's no built-in merge function. To combine sites, export data from both, manually consolidate in a spreadsheet, then recreate in a new site. For significant data volumes, consider if maintaining separate sites better serves your management needs.

Use the "Export All Data" button in Settings → Data Management. This creates a JSON file containing ALL sites and their data. Store this file securely. To restore, you would need to manually re-import (future feature).

Troubleshooting

Multi-Site Specific Issues

Assessments Not Showing in Correct Site
  • Verify you're viewing the correct site using the site selector
  • Check that the assessment was created while that site was selected
  • Open the assessment details to confirm the site ID
Species Missing from Dropdowns
  • Ensure you're in the correct site
  • Verify species were created for this site, not another
  • Check species type matches dropdown type (fauna vs flora)
Calendar Shows Wrong Activities
  • Calendar always shows activities for current site only
  • Refresh calendar after switching sites
  • Check activity site assignment in details view

General Issues

Data Not Saving
  • Check that all required fields are completed
  • Ensure browser has localStorage enabled
  • Try refreshing the page
  • Check storage quota (Settings → System Information)
Map Not Loading
  • Verify internet connection (map tiles require internet)
  • Check if mapping services are blocked
  • Refresh the page

Performance Tips

  • Limit photos per record to maintain performance
  • Export and archive old data periodically
  • Use table filtering for large datasets
  • Consider splitting very large sites into smaller ones
Support: For persistent issues, export your data and contact your system administrator. Include the export file when reporting bugs.
